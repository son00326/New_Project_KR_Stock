## 1. analyst 리뷰 기반 7개 Open Questions — 결정 확정

| # | Open Question | 확정 결정 | 근거 |
|---|---|---|---|
| Q1 | 5질문 = 섹션 오버레이 vs 재배치? | **오버레이** (기존 §0~§8 구조 유지, 질문 태그만 부착) | P1~P8 확정 구조 보존. 독자가 순차 읽으면 ①→⑤ 자연스럽게 도달 |
| Q2 | 3문장 요약 ↔ §0 ↔ Summary 표 계층? | **3계층**: (1) 3문장 요약(맨 위, 판정 결론까지) (2) Summary 표(6행, 지표·TP·투표) (3) §0 투자 서사(3문장 확장, "왜·왜 지금" 배경) | 정보 계층화 — 점진적 상세화 |
| Q3 | 섹터별 질문 순서 가변? | **5질문 고정** | 독자 학습 곡선 유지. 섹터 바뀌면 질문 자체가 아닌 "질문에 답하는 증거"만 바뀜 |
| Q4 | 오해 박스 출처·검증? | **writer 창작 허용 + document-specialist 후속 검증**. 현재는 섹션당 **1~2개** (27개→15개 상한) | analyst 우려 수용 (고밀도 피로) |
| Q5 | Red Flag Tier 1 위치? | **Summary 최상단 🚨 배너 + 질문⑤ 판정 안 중복**. Tier 1 발동 시 ①~④는 참고 섹션으로 강등 표시 | 자동 SELL 독자 우선 인지 |
| Q6 | 5질문 구어체 노출? | **허용**. "이 회사 뭐 하는데?" 구어체 그대로 노출 | 독자 = 본인+공동창업자+초대 500인, 격식 불필요 |
| Q7 | SELL/HOLD 서사 플립 규칙? | 질문② 라벨 가변: BUY 템플릿 = "왜 지금 주목?" / SELL = "왜 지금 조심?" / HOLD = "왜 지금 관망?". 템플릿 공통, 라벨만 교체 | 편향 방지 |

### 추가 분량·형식 상한 (analyst 권고 수용)

| 컴포넌트 | 상한 | 비고 |
|---|---|---|
| 섹션 오프닝 템플릿 | **1문장 + 불릿 3개** | "이 섹션에서는 [질문]에 답하기 위해 [증거 3개]를 봅니다" |
| Transition 문장 | **2문장 / 40단어 이내, 섹션 도입부만** | 이전 섹션 결론 + 다음 질문 |
| 오해 박스 | **각 3줄 이내** (질문 1줄 + 교정 2줄) | 섹션당 1~2개 |
| 3문장 요약 | **각 1문장 80자 이내** | 판정·이유·리스크 1문장씩 |
| 본문 분량 | **C4 준수: 본문 400~500줄** | Appendix 별도 800줄 |
| 숫자 밀도 | **1 단락당 숫자 ≤3개** | 초보 가독성 |

## 2. 5질문 → 섹션 매핑 표 (오버레이 방식)

| 질문 | 구어 라벨 (BUY 기준) | 주 섹션 | 보조 섹션 | 독자가 얻는 것 |
|---|---|---|---|---|
| ① | 이 회사 뭐 하는데? | §0 투자 서사 + §1 산업 기초 | §2 회사 개요(히스토리·경영진) | 사업 모델·업황의 1문단 이해 |
| ② | 왜 지금 주목? (SELL 시 "왜 지금 조심") | §2 Track Record + §3 이익 모델 | §7 촉매 캘린더 | 타이밍의 근거 |
| ③ | 얼마가 적정가인데? | §4 밸류에이션 | §6 시나리오 TP | 공식 TP + 현재가 괴리 |
| ④ | 뭐가 틀어지면 안 되나? | §5 매크로 + §7 리스크 매트릭스 | §4 Invalidation 조건 | 트리거·손절선 |
| ⑤ | 살까 말까? | §8 투심위 + Summary | §6 Pre-Mortem | 최종 판정 + 포지션 크기 |

### 매핑 셀 완결성 (analyst Acceptance Criteria #2 충족)

45셀 (9섹션 × 5질문) 완결. 주/보조 라벨 모두 기재.

### SELL·HOLD·Red Flag 분기 (analyst Edge Cases 1~3)

- **BUY**: 위 5질문 기본 라벨
- **SELL**: 질문② 라벨만 플립 ("왜 지금 조심"). 나머지 구조 동일. Pre-Mortem이 Bull Case와 비중 역전
- **HOLD**: 질문② = "왜 지금 관망". 질문⑤ = "언제 재평가?" + Invalidation 양방향 조건 강조 (P4-2 준수)
- **Red Flag Tier 1 자동 SELL**: Summary 🚨 배너 최상단. ①~④ 섹션은 "참고 정보"로 강등 안내. 판정은 §8 단축판으로 즉시 제공

## 3. 섹션 오프닝 템플릿 (1문장 + 불릿 3개 고정)

```
[질문] 에 답하기 위해, 이 섹션에서는 다음 3가지를 봅니다.
- 증거 1: [핵심 지표명 + 한 줄 요약]
- 증거 2: [핵심 지표명 + 한 줄 요약]
- 증거 3: [핵심 지표명 + 한 줄 요약]
```

### 예시 (§4 밸류에이션, BUY)

```
③ "얼마가 적정가인데?"에 답하기 위해, 이 섹션에서는 다음 3가지를 봅니다.
- 증거 1: 비교 가능한 회사 딱 1곳 (Halozyme) 과의 배수 격차
- 증거 2: 2028년 예상 매출 기준으로 계산한 적정가
- 증거 3: 증권사 9개 기관의 목표가 분포
```

## 5. "3문장 요약" 구조 (Summary 최상단 고정)

```
[1문장] 이 회사는 [한 줄 사업 모델]이고, [섹터 비유]에 해당합니다.
[2문장] [판정: BUY/HOLD/SELL] — 이유는 [핵심 근거 1개]입니다.
[3문장] 단, [핵심 리스크 1개]가 현실화되면 판단이 뒤집힙니다.
```

### 예시 (알테오젠, BUY 가정)

```
[1] 알테오젠은 주사제를 정맥→피하 방식으로 바꾸는 기술을 빅파마에 파는 회사로,
    건설업의 "공법 특허를 대형 건설사에 라이선스하고 로열티 받는 모델"에 해당합니다.
[2] BUY — 이유는 2025년 매출이 전년 대비 2.1배 늘었고, 키트루다SC 로열티
    (2043년까지 2%)가 이제 막 들어오기 시작했기 때문입니다.
[3] 단, "로열티율 2%가 이후 계약에서 5%로 오르지 못하면" 현재 주가는 비싼
    수준으로 재평가됩니다.
```

## 6. "흔히 하는 오해" 박스 — 공통 20개 + 섹터별 7~10개 분리 (analyst Recommendation #4)

### 형식

```
💡 흔히 하는 오해
"[초보 질문 1줄]"
→ 실제로는 [교정 2줄 이내]
```

### 공통 오해 풀 (전 섹터, 20개 초안)

1. PER 높으면 무조건 비싼 주식? → 성장률이 PER 배수를 정당화하면 합리 구간
2. PSR 높으면 고평가? → 로열티·플랫폼 회사는 매출 자체가 작아서 자연 상승
3. 목표가 높으면 좋은 종목? → 목표가 ≠ 보장가. 시나리오 확률에 따른 기대값이 핵심
4. 증권사 BUY 전원 일치면 안전? → 동질 집단의 편향. 실패 사례일수록 BUY 유지됨
5. 최대 손실(Max DD) 낮으면 안전? → 기간 짧으면 의미 없음. 사이클 1회 이상 포함 필수
6. 배당 높으면 좋은 회사? → 배당 성향 >100%면 재무 건전성 훼손 경고
7. 현금 많으면 안전? → 쓸 곳 없으면 자산 효율 저하. ROE 같이 봐야 함
8. 매출 성장률 높으면 좋은 회사? → 적자 성장은 현금 소진. 영업이익률 병행 확인
9. 경영진 지분 많으면 안심? → 담보 대출 여부·자녀 증여 시점 병행 확인
10. 외국인 매수 = 매수 신호? → 패시브 ETF 리밸런싱 매수일 수 있음
11. 거래량 급증 = 매수 신호? → 분산 매도 신호일 가능성 동등
12. 기술적 지표(이동평균) 상향 돌파 = BUY? → 후행 지표. 펀더멘털 변화 선행 필수
13. 특허 많으면 Moat? → 핵심 특허 1개 vs 주변 특허 100개는 가치 다름
14. CEO 인터뷰 자신감 = 실현 확률? → IR 편향. Track Record로 검증
15. 섹터 1등 = 안전? → "안 죽는 회사"일 뿐, 초과 수익 기대와는 다른 축
16. 배당수익률 높으면 Bear 방어? → 실적 악화 시 배당 컷이 먼저 옴
17. 레버리지 ETF = 2배 수익? → 장기 보유 시 음의 복리로 수익 잠식
18. ESG 점수 높으면 리스크 낮음? → 특정 기관 기준. 섹터 평균 대비 편차 확인
19. 공매도 많으면 하락 예정? → 공매도 수급 기술적 반등 트리거 가능
20. 분할 매수 = 손실 축소? → 추세 반전 없으면 평단가 낮아져도 절대 손실 증가

### 바이오 섹터별 오해 (7개 초안)

B1. 임상 3상 진입 = 승인 임박? → 3상 성공률 50% 내외
B2. FDA 승인 = 매출 보장? → 보험 수가 협상 실패 시 판매 0
B3. 블록버스터 신약 특허 20년 보장? → 실제 10~14년 독점 (물질특허 vs 제법특허)
B4. 바이오시밀러 = 복제약? → 생물학적 동등성 증명 필요, 개발비 일반 복제약 10배
B5. 글로벌 빅파마 파트너십 = 성공? → 3건 중 2건은 중도 해지
B6. 로열티율 높으면 좋은 딜? → 업프론트·마일스톤과 트레이드오프
B7. SC 전환 기술 모든 약에 적용? → 대분자 중 일부만. 분자량·농도 제약

### 건설업 독자 번역 매핑 (예시)

- 공통 #3 "목표가 보장 아님" = 건설업의 "분양가 고시 ≠ 실제 분양률"
- 공통 #8 "매출 성장 자체가 선?" = 건설업의 "수주 잔고 늘어도 원가율 악화 시 적자"
- B6 "로열티 높으면 좋은 딜?" = 건설업의 "도급률 높아도 하자보수 충당금으로 상계"

## 7. 5질문 × 알테오젠 적용 prose 예시 (writer 참조, 총 25줄 상한)

### Q1. "알테오젠 뭐 하는 회사인데?"
알테오젠은 "정맥주사제(병원에서 30분 꽂고 있어야 하는 약)를 피하주사(2분 만에 끝)"로 바꾸는 기술을 대형 제약사에 라이선스하고 로열티를 받는 회사입니다. 건설업으로 치면, "입주민 이주 없이 할 수 있는 리모델링 공법 특허"를 대형 건설사에 팔고 준공 후 관리비의 일부를 수취하는 구조와 유사합니다.

### Q2. "왜 지금 주목해야 해?"
2025년 9월 MSD의 키트루다SC(항암제)가 FDA 승인을 받으면서 알테오젠의 첫 로열티 스트림이 시작됐습니다. 이는 10년 R&D가 상업화 단계로 전환된 첫 사례입니다. 추가로 2024~2026년 3년간 AZ·다이이찌·GSK·바이오젠 등과 5건의 새 라이선스를 체결해 파이프라인이 8건으로 확장됐습니다.

### Q3. "얼마가 적정가?"
공식 목표가는 **430,000원**(현재가 361,500원 대비 +18.9%). 비슷한 회사 Halozyme 대비 매출 배수가 2.4배 비싼 편이지만, 알테오젠의 매출 성장률(+317%)이 Halozyme(+40%)의 8배라 프리미엄이 정당합니다. 3년 뒤 2028년 매출 기준으로 보면 배수 격차는 1.5배로 좁혀집니다.

### Q4. "뭐가 틀어지면 안 되나?"
핵심 3개: (1) 추후 딜의 **로열티율이 5% 이상으로 확인**되지 않으면, (2) 키트루다의 **2028년 물질특허 만료**에 대응한 후속 파이프라인 없으면, (3) 환율이 **달러 1,200원 아래**로 하락하면 (매출 100% 달러 기반). 이 셋 중 하나라도 발생 시 TP는 250,000원까지 하향.

### Q5. "살까 말까?"
투심위 10인 **BUY 7 / HOLD 2 / SELL 1**. 공식 판정 **BUY, 포지션 10%** (Half Kelly 기준). 단, Tier 1 Red Flag 없음 확인 완료. Invalidation 트리거 자동 모니터링 대상.

## 8. writer 프롬프트에 박제할 금지 사항 (analyst Guardrails 수용)

### 본문 등장 금지 표현
- "Peer 5축 필터링을 적용하면…" / "Pure-play vs Secondary 분리 시…" — 메타 분석 용어
- "Investment Thesis Bridge 1/2/3" — 내부 설계 용어
- "Moat Type: Intangible" — 영어 원문 그대로. "해자(경쟁사가 따라올 수 없는 벽)"로 선행 비유 후 원문 병기
- PSR/PER/WACC/EV/EBITDA/DCF/LTM/NTM 용어 **첫 등장 시 비유 없이 사용**

### 표 사용 제약
- 본문 표 **3열 이하**
- 4열 이상은 **Appendix로 이동**
- 1 섹션 표 **최대 2개**

### 문단 구조
- 1 단락 = 3~5문장
- 1 단락 숫자 = ≤3개
- 연속 표 2개 금지 (사이에 prose 1 단락 필수)

### 12.4 §8 투심위 서사 규격 (v2 §8 그대로 복원)

**Part A. Sector Board 투표** (국내 5명 + 해외 5명)

- 투표 결과 집계 표 (3열: 위원 / 의견 / 투표)

**Part B. Core Committee 쟁점 토론**
- 쟁점 2~3개 도출 (예: "로열티율 2%가 치명적인가" / "특허 만료 2028 리스크")
- 쟁점별:
  - 찬성 대표 발언 (prose 3~4문장)
  - 반대 대표 발언 (prose 3~4문장)
  - 중재자 정리 (1~2문장)

**Part C. 최종 판정**
- 투표 집계 (BUY N / HOLD N / SELL N)
- 공식 판정 + 근거 2~3줄
- Co-Chair 만장일치 여부
- Position Size (Half Kelly 기반 %)

**Part D. Invalidation 조건 (§4와 연동)**
- 3요소: 가격 트리거 / 이벤트 트리거 / 기한
- BUY 판정일 때도 SELL 플립 조건 명시
