## 1. 핵심 미션

> **"자기가 매일 쓸 수 있는 최고 품질의 주식 분석 플랫폼을 만들고,
> 본인 자본으로 실전 검증하며,
> 신뢰 네트워크에 법적 안전하게 배포하는 듀얼 트랙 프로젝트"**

**본질**: 플랫폼 그 자체 (개인용 + 지인 배포)
**부가 가치**: 본인 자금 운용을 통한 트랙 레코드 구축
**최악 시나리오**: 개인 투자 도구로 계속 사용 (사업 실패 개념 자체가 없음)

## 3. 사업 구조 — 3-Layer Architecture

```
┌─────────────────────────────────────────┐
│  Layer 1: 플랫폼 그 자체 (본질)           │
│  - 개인이 매일 쓸 수 있는 최고 품질 도구  │
│  - 동의서 기반 제3자 배포                │
│  - 법적 프레임: "정보·분석 도구"          │
│  - 성공 지표: 플랫폼 품질, 배포 규모     │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  Layer 2: 실전 검증 (운용)                │
│  - 본인 자금 15억, 공동창업자 별도 자금   │
│  - 각자 독립 운용 (Dual Trader)          │
│  - Y1 트랙 레코드 구축                   │
│  - 법적 프레임: "개인 투자"               │
│  - 성공 지표: 수익률, Sharpe, KOSPI alpha │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  Layer 3: 확장 (선택적)                  │
│  - Y1 성과 기반 동적 결정                 │
│  - 가족 자본 확대 → 친지·지인 LP         │
│  - 등록 경로: 자문업 → 사모펀드          │
│  - 성공 지표: AUM, 수수료 수익           │
└─────────────────────────────────────────┘
```

**각 Layer의 독립적 가치**:
- Layer 1만 성공 → "배포 가능한 고품질 분석 도구"
- Layer 1+2 성공 → "트랙 레코드 있는 자기 운용자"
- Layer 1+2+3 성공 → "부티크 자산운용사"

## 4. 확정된 기획 사항 (Q1~Q11)

### Q1. 초기 운용 규모
- 시작: 5~10명 이내 (본인 중심)
- 점진 확장: 성과 기반
- 법적 리스크: 최소 (특정 다수)

### Q2. 초기 AUM
- **본인 자금 15억 100%** (자기 자본)
- 성과 검증 후 **부모님 자본 추가 투입** 고려
- 외부 LP 유치는 Y1 말 이후

### Q3. 목표 최종 규모
- **유보** (플랫폼 output에 따라 동적 결정)
- 후보 시나리오:
  - 보수: AUM 50~100억 (부티크 프롭 트레이더)
  - 현실: AUM 200~400억 (가족·지인 LP)
  - 야심: AUM 500~1,000억 (정식 등록 부티크 자산운용사)

### Q4. 법적 등록 의사
- **Y1 말 결정** (12개월 트랙 레코드 보고)
- 사전 Decision Tree 설정:
  - KOSPI +15% 이상 → 전문사모 집합투자업 (자본금 7.5억)
  - KOSPI +5~15% → 투자자문업 (자본금 1~2.5억)
  - KOSPI ±5% → 등록 유보, Y1.5 연장
  - KOSPI -5% 이하 → 재검토 or 운용 포기

### Q5. 수수료 모델 (Y2+ 지인/LP 확장 시)
- **2/20 모델** — 관리 2% + 성과 20%
- High Water Mark 필수 (이전 최고점 기준)
- Hurdle Rate: KOSPI 또는 연 5% 고정
- Y0~Y1은 본인 자금이므로 수수료 0

### Q6. 생존 자금
- **사용자**: 월 생활비 150만원, 1년치 여유 + 운용 자산 중 최대 2억 비상 인출 가능
- **공동창업자**: 생활비 압박 없음 (별도 수입원)
- **결론**: 12개월 집중 몰입 가능, 재무 압박 0

### Q7. 운영비 & 자본금
- **월 운영비**: 100만원 MAX
  - 인프라(Vercel, Supabase): 15만
  - AI API(Claude): 20~40만
  - 법무·세무: 15~20만
  - 기타: 10만
- **등록 자본금**: Y1 말 다양한 방식 고려 가능
  - Hybrid 구조 권장: 개인 자산 + 필요 시 확대

### Q8. 실패 대비 & 마인드셋
- **유연파** ("실패도 데이터")
- **재취업 안전판 있음** (육아휴직 중)
- **최우선 목표**: 개인용으로도 사용할 퀄리티의 플랫폼 구축
- **실패 정의 없음**: 사업 실패 시에도 개인 도구로 계속 사용

### Q9. 외부 투자
- **완전 자력 (Bootstrapping)**
- 엔젤·VC 유치 불필요 (지분 희석 목적 없음)
- 운용 자금 확대 필요 시: **부모님 자본** (Y1 성과 우수 시)

### Q9.5. 제3자 배포
- **Phase 1**: 지인 네트워크 30~100명 (Private)
- **Phase 2+**: 초대제 클로즈드 베타 100~500명 (성과 의존)
- **과금**: 소액~중간 유료 (월 1~20만원 사이, 구체 결정 유보)
- **배포 시점**: Y1 중반 (6개월) 목표
- **안전 임계치**: 500명 이하 유지 (특정 다수)
- **법적 장치**: 이용약관 + 면책 동의서 + 정보 제공 프레임

### Q10. 역할 분화 — Dual Trader Architecture
- **운영비**: 5:5 공동 부담
- **운용 자금**: 각자 독립 관리 (사용자 15억, 공동창업자 ?억)
- **운용 의사결정**: 각자 독립 (Y1 병렬 검증)
- **플랫폼 1차 설계**: 사용자 단독
- **플랫폼 고도화**: 공동 투입
- **공동 운영**: Y1 성과에 따라 추후 결정
- **R&R 정의**: git push 후 조만간 확정 예정

### Q11. 지분 및 과금 원칙
- **지분**: 5:5 유지 (Vesting 논의 유보)
- **지분 논의 타이밍**: 플랫폼 품질 안정화 후
- **과금 시점**: 지인 확장 단계부터 (M3-6 이후)
- **과금 철학**: **이익 최대화 아닌, 서비스 비용 충당 수준**
  - 권장 가격: 월 19,900원 (부담 없음 + 비용 충당 + 지속 가능)
  - 운영비 50~80만 + 버퍼 20만 = 월 70~100만 매출 필요

### 8.2 투심 보고서 프레임워크 (Section 0~8 + Appendix, v2.0)

> 상세 가이드라인: `Document/ReportFramework.md` 참조

| Section | 내용 |
|---------|------|
| Summary | 1페이지 요약 (핵심 숫자 + 투심위 결론 + 시나리오 TP + 리스크 3가지) |
| 0. Investment Narrative | **[신규]** 투자 서사 — "무엇을/왜/왜 지금" 비전문가 설명 |

| 2. Company Overview | **[신규]** 회사 개요 — 히스토리, 경영진, 지분 구조, 핵심 기술 비유 |
| 3. Earnings Model | 이익 모델 — 수익 구조 비유, 과거 실적, 컨센서스, 3~5년 로드맵 |

| 6. Scenario Analysis | 시나리오 분석 — Bull/Base/Bear + Pre-Mortem 3가지 |
| 7. Catalyst & Risk | **[통합]** 촉매+리스크 — 캘린더, 리스크 매트릭스, 기술적 지표 핵심 4~5개만 |
| 8. Committee Votes | 투심위 — **쟁점별 찬반 토론** + 최종 투표 |
| Appendix | **[신규]** 기술적 지표 상세, 재무제표 상세, DCF 상세, 용어 해설 |

- 산업 기초를 Section 1로 앞으로 이동 → 산업 이해 선행

**보고서 형식**: Summary 1페이지 + Section 0~8 In-depth + Appendix

### 8.3 투심위 토론 구조 (v2.0)

**변경: "각자 독백" → "쟁점별 찬반 대결"**

```
Part A: 섹터 보드 (10명)
  Step 1: 각 위원 독립 의견 (BUY/HOLD/SELL + TP + 논거)
  Step 2: [신규] 핵심 쟁점 2~3개 도출
  Step 3: [신규] 쟁점별 찬성 대표 vs 반대 대표 vs 중재자 토론
  Step 4: 최종 집계

Part B: Core Committee (10명 + 위원장)
  Step 1: 섹터 보드 결과 수신
  Step 2: 독립 의견 (투자 철학 반영 강제)
  Step 3: [신규] 핵심 쟁점 2차 토론
  Step 4: 최종 투표 + 종합
```

**위원 선정 원칙**:
- 독립성: 동일 논거 2명 이상 사용 금지. 같은 결론이라도 근거가 달라야 함.
- 철학 반영: Core 각 위원은 자신의 투자 철학에서 자연스럽게 나오는 관점만 사용.
- 고유 통찰: 섹터 보드 위원은 전문 분야 고유 통찰 1개 이상 필수.
- 면책: "실명은 투자 스타일 시뮬레이션 목적, 실제 인물 의견 아님" 필수 표기.

### 8.4 Core Committee 투자 철학 매핑 (확정)

| 슬롯 | 투자 철학 | 인물 | 핵심 관점 |
|------|----------|------|----------|

| 2 | 역발상/사이클 투자 | Howard Marks | "시장이 과도하게 비관적인가, 리스크 프리미엄이 적정한가" |

## 10. Manual 투자 트랙 (확정)

### 10.1 개요

AI가 KOSPI/KOSDAQ 전종목(~2,500개)을 분석하여 upside가 높은 종목을 **Short List(30종목)**로 추천. 유저는 리스트만 확인하고, 납득되면 매수로 연결하는 구조.

```
전종목 2,500개
  │
  ▼ [업황 사이클 기반 2단계 스크리닝]
  │  ├── 업황 UP/개선 섹터 → 강화 스크리닝 (더 많은 종목 통과)
  │  ├── 업황 평탄 섹터 → 기본 경량 스크리닝
  │  └── 업황 DOWN 섹터 → 최소한만 통과
  │
  ▼ Top 50~100개
  │
  ▼ [AI Section 0~8 풀 분석 + 투심위 시뮬레이션]
  │
  ▼ Short List 30개 (단기 10 / 중장기 10 / 장기 10)
     → 유저에게 추천
```

### 10.2 Short List 구조

| 항목 | 설정 |
|------|------|
| **규모** | 단기 10개 + 중장기 10개 + 장기 10개 = 30개 |
| **중복** | 동일 종목이 복수 관점에 편입 가능 |
| **홀딩 기간** | 단기 1~3개월 (모멘텀 위주) / 중장기 3~12개월 / 장기 1년+ |
| **섹터 제약** | 없음 — 매크로에 따른 섹터 쏠림 납득 |
| **우선순위** | 예상 수익률 기반 종합 평가 (밸류에이션/매크로/테크니컬/펀더멘탈) |
| **편입 의견** | BUY만 (추천 리스트이므로 HOLD/SELL 불포함) |
| **접근 권한** | 어드민 전용 (본인+공동창업자), 권한 부여 가능 |

### 10.3 종목 선정 프로세스

**업황 판단 (AI 자동)**:
- AI가 전문가 수준으로 매크로 상황 분석 → 수혜 섹터/업황 매핑
- 단순 리서치 X, 면밀한 검토 필수 (시스템 핵심 경쟁력)
- 업황 사이클 판정: UP / 개선 중 / 평탄 / DOWN → 스크리닝 강도 차등

**스크리닝**:
- 최소 진입 기준: 별도 하한선 없음, 스크리닝 과정에서 자연 탈락
- 결과: 50~100개 후보

**풀 분석 → 최종 선정**:
- 50~100개 후보에 Section 0~8 풀 분석 + 투심위 시뮬레이션 적용
- 예상 수익률 종합 평가로 순위 매김 → 관점별 상위 10개 = Short List 30개

### 10.4 리포트 3-Tier 체계

| Tier | 대상 | 내용 | 생성 | 비용 |
|------|------|------|------|------|
| **Tier 1** | Short List 30개 | 풀 Section 0~8 + 투심위 | 사전 생성 (월간) | ~12만/월 |
| **Tier 2** | 스크리닝 상위 50~100개 | 풀 Section 0~8 + 투심위 | 사전 생성 (월간) | ~17만/월 |
| **Tier 3** | 나머지 전종목 | 정량 프로필 카드 (기존 tudal 종목 4탭 수준 이상) | 데이터 자동 산출 | ~0 |

Tier 3 최소 기준 = 가격 차트 + 기업 개요 + 재무 요약 + 기술적 지표 + 수급

### 10.5 유저 접근 방식

**Short List (패시브)**: 목록 화면에서 종목명 + 현재가 + TP + Upside% + 투심위 의견 + 핵심 촉매 확인 → 클릭 시 풀 리포트(Section 0~8) 연결

**검색 (액티브)**: Tier 1/2 = 풀 리포트 열람, Tier 3 = 정량 프로필 카드 열람

### 10.6 리밸런싱 & 변동 관리

| 항목 | 설정 |
|------|------|
| 기본 주기 | 월간 (홀딩 필요 시 분기) |
| 변동 표시 | 신규 편입 / 유지 / 제외 + 변동 사유 명시 |
| 리포트 갱신 | 월간 기본 + 실적 발표/대형 이벤트 시 수시 갱신 |
| 긴급 변동 | 즉시 Short List 제외/교체 + 이메일/텔레그램 알림 |
