## 1. 핵심 미션

> **"자기가 매일 쓸 수 있는 최고 품질의 주식 분석 플랫폼을 만들고,
> 본인 자본으로 실전 검증하며,
> 신뢰 네트워크에 법적 안전하게 배포하는 듀얼 트랙 프로젝트"**

**본질**: 플랫폼 그 자체 (개인용 + 지인 배포)
**부가 가치**: 본인 자금 운용을 통한 트랙 레코드 구축
**최악 시나리오**: 개인 투자 도구로 계속 사용 (사업 실패 개념 자체가 없음)

## 3. 사업 구조 — 3-Layer Architecture

```
┌─────────────────────────────────────────┐
│  Layer 1: 플랫폼 그 자체 (본질)           │
│  - 개인이 매일 쓸 수 있는 최고 품질 도구  │
│  - 동의서 기반 제3자 배포                │
│  - 법적 프레임: "정보·분석 도구"          │
│  - 성공 지표: 플랫폼 품질, 배포 규모     │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  Layer 2: 실전 검증 (운용)                │
│  - 본인 자금 15억, 공동창업자 별도 자금   │
│  - 각자 독립 운용 (Dual Trader)          │
│  - Y1 트랙 레코드 구축                   │
│  - 법적 프레임: "개인 투자"               │
│  - 성공 지표: 수익률, Sharpe, KOSPI alpha │
└─────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────┐
│  Layer 3: 확장 (선택적)                  │
│  - Y1 성과 기반 동적 결정                 │
│  - 가족 자본 확대 → 친지·지인 LP         │
│  - 등록 경로: 자문업 → 사모펀드          │
│  - 성공 지표: AUM, 수수료 수익           │
└─────────────────────────────────────────┘
```

**각 Layer의 독립적 가치**:
- Layer 1만 성공 → "배포 가능한 고품질 분석 도구"
- Layer 1+2 성공 → "트랙 레코드 있는 자기 운용자"
- Layer 1+2+3 성공 → "부티크 자산운용사"

## 4. 확정된 기획 사항 (Q1~Q11)

### Q1. 초기 운용 규모
- 시작: 어드민(본인)이 메인 서비스(Short List 30) 운용. 멤버 페이지(랜딩→로그인→메인)는 `Document/Research/` 사례 형식으로 별도 제공
- 점진 확장: 멤버 cap 500명·초대 전용 유지, 성과 기반으로 단계별 온보딩
- 법적 리스크: 최소 (특정 다수)

### Q2. 초기 AUM
- **본인 자금 15억 100%** (자기 자본)
- 성과 검증 후 **부모님 자본 추가 투입** 고려
- 외부 LP 유치는 Y1 말 이후

### Q3. 목표 최종 규모
- **유보** (플랫폼 output에 따라 동적 결정)
- **본 시점 메인 서비스는 어드민(본인) 운용으로 한정** — AUM 50~1,000억 시나리오는 **선택적 Layer 3 시나리오 (Y2+ 검토 예정)**
- 후보 시나리오 (Layer 3):
  - 보수: AUM 50~100억 (부티크 프롭 트레이더)
  - 현실: AUM 200~400억 (가족·지인 LP)
  - 야심: AUM 500~1,000억 (정식 등록 부티크 자산운용사)

### Q4. 법적 등록 의사
- **Y1 말 결정** (12개월 트랙 레코드 보고)
- 사전 Decision Tree 설정:
  - KOSPI +15% 이상 → 전문사모 집합투자업 (자본금 7.5억)
  - KOSPI +5~15% → 투자자문업 (자본금 1~2.5억)
  - KOSPI ±5% → 등록 유보, Y1.5 연장
  - KOSPI -5% 이하 → 재검토 or 운용 포기

### Q5. 수수료 모델 (Y2+ 지인/LP 확장 시)
- **2/20 모델** — 관리 2% + 성과 20%
- High Water Mark 필수 (이전 최고점 기준)
- Hurdle Rate: KOSPI 또는 연 5% 고정
- Y0~Y1은 본인 자금이므로 수수료 0

### Q6. 생존 자금
- **사용자**: 월 생활비 150만원, 1년치 여유 + 운용 자산 중 최대 2억 비상 인출 가능
- **공동창업자**: 생활비 압박 없음 (별도 수입원)
- **결론**: 12개월 집중 몰입 가능, 재무 압박 0

### Q7. 운영비 & 자본금
- **월 운영비**: 100만원 MAX
  - 인프라(Vercel, Supabase): 15만
  - AI API(Claude): 20~40만
  - 법무·세무: 15~20만
  - 기타: 10만
- **등록 자본금**: Y1 말 다양한 방식 고려 가능
  - Hybrid 구조 권장: 개인 자산 + 필요 시 확대

### Q8. 실패 대비 & 마인드셋
- **유연파** ("실패도 데이터")
- **재취업 안전판 있음** (육아휴직 중)
- **최우선 목표**: 개인용으로도 사용할 퀄리티의 플랫폼 구축
- **실패 정의 없음**: 사업 실패 시에도 개인 도구로 계속 사용

### Q9. 외부 투자
- **완전 자력 (Bootstrapping)**
- 엔젤·VC 유치 불필요 (지분 희석 목적 없음)
- 운용 자금 확대 필요 시: **부모님 자본** (Y1 성과 우수 시)

### Q9.5. 제3자 배포
- **Phase 1**: 지인 네트워크 30~100명 (Private)
- **Phase 2+**: 초대제 클로즈드 베타 100~500명 (성과 의존)
- **중요 — 노출 범위**: 메인 서비스(Short List 30·풀 리포트)는 **어드민 전용**으로 운용. 멤버 100~500명에게는 **멤버 페이지(Research 형식)만 노출**. Short List·Tier 1/2 풀 리포트는 멤버에게 노출하지 않음.
- **과금**: 소액~중간 유료 (월 1~20만원 사이, 구체 결정 유보)
- **배포 시점**: Y1 중반 (6개월) 목표
- **안전 임계치**: 500명 이하 유지 (특정 다수)
- **법적 장치**: 이용약관 + 면책 동의서 + 정보 제공 프레임

### Q10. 역할 분화 — Dual Trader Architecture
- **운영비**: 5:5 공동 부담
- **운용 자금**: 각자 독립 관리 (사용자 15억, 공동창업자 ?억)
- **운용 의사결정**: 각자 독립 (Y1 병렬 검증)
- **플랫폼 1차 설계**: 사용자 단독
- **플랫폼 고도화**: 공동 투입
- **어드민 권한**: 어드민(사용자 본인) 1명 고정. 공동창업자는 **운영 파트너 역할**(자금·운영비 공동 부담, 자기 자금 독립 운용)이며, 메인 서비스(Short List 30·풀 리포트) 어드민 로그인 권한은 본인에게만 부여.
- **공동 운영**: Y1 성과에 따라 추후 결정
- **R&R 정의**: git push 후 조만간 확정 예정

### Q11. 지분 및 과금 원칙
- **지분**: 5:5 유지 (Vesting 논의 유보)
- **지분 논의 타이밍**: 플랫폼 품질 안정화 후
- **과금 시점**: 지인 확장 단계부터 (M3-6 이후)
- **과금 철학**: **이익 최대화 아닌, 서비스 비용 충당 수준**
  - 권장 가격: 월 19,900원 (부담 없음 + 비용 충당 + 지속 가능)
  - 운영비 50~80만 + 버퍼 20만 = 월 70~100만 매출 필요

## 5. 단계별 실행 로드맵

### Stage 1: Private Alpha (M0-3)
- 대상: 창업자 2인 본인 사용
- 가격: 무료 (→ 정식 가격은 §Q11 참조)
- 범위: **어드민 기능만** (Short List 30·풀 리포트). 멤버 노출 기능 없음.
- 작업:
  - 기존 코드베이스 Option B 기준 간소화
  - 실데이터 연결 (KRX, 한투 OpenAPI, DART)
  - 본인 15억 운용 시작
  - 분석 프레임워크 완성

### Stage 2: Friends & Family Beta (M3-6)
- 대상: 지인 5~15명 선별
- 가격: 무료 (피드백 교환)
- 범위: **어드민 기능**(본인 전용) + **멤버 페이지 초기본**(Research 형식) 노출 시작. Short List·풀 리포트는 비공개.
- 작업:
  - 초기 멤버 경험 검증
  - UX 개선
  - 동의서 초안 작성
  - 법무 자문 1차 (50만 스팟)

### Stage 3: Closed Beta (M6-12)
- 대상: 지인 30~100명
- 가격: 월 19,900원 (→ §Q11)
- 범위: **멤버 페이지(Research 형식) 유료 전환**. 메인 서비스(Short List 30·풀 리포트)는 여전히 어드민 전용.
- 작업:
  - 초대 기반 배포
  - 유료 전환 (월 매출 60~200만)
  - 트랙 레코드 6~12개월 누적
  - Y1 말 Decision Tree 적용

### Stage 4+: Y1 말 이후 동적 결정
- 범위: 멤버 cap 500명 한도 내에서 멤버 페이지 확대. 어드민 전용 범위 유지.
- Output 기반으로 확장 방향 결정
- 법적 등록 여부 결정
- 부모님 자본 투입 여부 결정
- 초대제 100~500명 확장 여부 결정

## 9. 자동매매 트랙 — 3축 Quant 시스템

> **[미확정 — 기준선 단순화 후 재검토 필요]**
> 본 §9는 자동매매 트랙(리포트와 독립된 외적 트랙)의 리서치 초안. 기준선(어드민 메인 서비스 = Short List 30·풀 리포트)이 리포트 트랙 중심으로 단순화됨에 따라, §9.1~§9.4의 3축 배분·Quant Board·백테스트 결과는 자동매매 트랙이 정식 착수되는 시점에 재검토·재확정한다. 현재 서술은 보존한다.

### 9.1 시스템 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    Early Warning System                       │
│  MA구조 + 모멘텀다이버전스 + 변동성 + 거래량 + RSI 급락        │
│  → 크래시 7~15일 전 선행 감지 → 점진적 디리스킹               │
├─────────────────────────────────────────────────────────────┤
│                    Crisis Management Layer                    │
│  Layer 1: Early Warning (선행) — 점진적 포지션 축소            │
│  Layer 2: Reactive (반응) — 크래시 즉시 10~25% 축소           │
│  → 복합 레짐 감지 (Bull/Sideways/Bear)                       │
├─────────┬───────────┬───────────────────────────────────────┤
│ 단기 30% │ 중기 40%   │ 장기 30%                              │
│ 21일 주기│ 42일 주기  │ 63일 주기                              │
│ 모멘텀   │ 균형 멀티  │ 밸류+퀄리티                            │
│ +수급    │ 팩터       │                                       │
│ KOSDAQ   │ KOSPI      │ KOSPI Top10                          │
│ 모멘텀   │ 상위 20    │ 대형 가치주                            │
└─────────┴───────────┴───────────────────────────────────────┘
           ↑ 종목 선정: 10명 Quant Board 투표 (70% 컨센서스)
```

## 10. Manual 투자 트랙 (확정)

### 10.1 개요

AI가 KOSPI/KOSDAQ 전종목(~2,500개)을 분석하여 upside가 높은 종목을 **Short List(30종목)**로 추천. **어드민(사용자 본인)이** 리스트를 확인하고, 납득되면 매수로 연결하는 구조. 멤버에게는 노출하지 않는다.

```
전종목 2,500개
  │
  ▼ [업황 사이클 기반 2단계 스크리닝]
  │  ├── 업황 UP/개선 섹터 → 강화 스크리닝 (더 많은 종목 통과)
  │  ├── 업황 평탄 섹터 → 기본 경량 스크리닝
  │  └── 업황 DOWN 섹터 → 최소한만 통과
  │
  ▼ Top 50~100개
  │
  ▼ [AI Section 0~8 풀 분석 + 투심위 시뮬레이션]
  │
  ▼ Short List 30개 (단기 10 / 중기 10 / 장기 10)
     → 어드민에게 추천
```

### 10.2 Short List 구조

| 항목 | 설정 |
|------|------|
| **규모** | 단기 10개 + 중기 10개 + 장기 10개 = 30개 |
| **중복** | 동일 종목이 복수 관점에 편입 가능 |
| **홀딩 기간** | 단기 1~3개월 (모멘텀 위주) / 중기 3~12개월 / 장기 1년+ |
| **섹터 제약** | 없음 — 매크로에 따른 섹터 쏠림 납득 |
| **우선순위** | 예상 수익률 기반 종합 평가 (밸류에이션/매크로/테크니컬/펀더멘탈) |
| **편입 의견** | BUY만 (추천 리스트이므로 HOLD/SELL 불포함) |
| **접근 권한** | 어드민(사용자 본인) 전용 |

### 10.3 종목 선정 프로세스

**업황 판단 (AI 자동)**:
- AI가 전문가 수준으로 매크로 상황 분석 → 수혜 섹터/업황 매핑
- 단순 리서치 X, 면밀한 검토 필수 (시스템 핵심 경쟁력)
- 업황 사이클 판정: UP / 개선 중 / 평탄 / DOWN → 스크리닝 강도 차등

**스크리닝**:
- 최소 진입 기준: 별도 하한선 없음, 스크리닝 과정에서 자연 탈락
- 결과: 50~100개 후보

**풀 분석 → 최종 선정**:
- 50~100개 후보에 Section 0~8 풀 분석 + 투심위 시뮬레이션 적용
- 예상 수익률 종합 평가로 순위 매김 → 관점별 상위 10개 = Short List 30개

### 10.4 리포트 3-Tier 체계

> **Tier 2/3는 어드민(사용자 본인)이 내부 파이프라인 산출물로만 참고**. 멤버·외부 노출 없음. Tier 1(Short List 30)도 어드민 전용.

| Tier | 대상 | 내용 | 생성 | 비용 | 노출 |
|------|------|------|------|------|------|
| **Tier 1** | Short List 30개 | 풀 Section 0~8 + 투심위 | 사전 생성 (월 1회 재선정) | ~12만/월 | 어드민 전용 |
| **Tier 2** | 스크리닝 상위 50~100개 | 풀 Section 0~8 + 투심위 | 사전 생성 (월 1회) | ~17만/월 | 내부 파이프라인(어드민 도구) |
| **Tier 3** | 나머지 전종목 | 정량 프로필 카드 | 데이터 자동 산출 | ~0 | 내부 파이프라인(어드민 도구) |

Tier 3 최소 기준 = 가격 차트 + 기업 개요 + 재무 요약 + 기술적 지표 + 수급

### 10.5 어드민 접근 방식

> 본 절 전체가 어드민(사용자 본인) 전용 동작이다. 멤버에게는 노출하지 않는다.

**Short List (패시브)**: 어드민 대시보드에서 종목명 + 현재가 + TP + Upside% + 투심위 의견 + 핵심 촉매 확인 → 클릭 시 풀 리포트(Section 0~8) 연결

**검색 (액티브, 어드민 내부 도구)**: Tier 1/2 = 풀 리포트 열람, Tier 3 = 정량 프로필 카드 열람

### 10.6 재선정 & 변동 관리

| 항목 | 설정 |
|------|------|
| 기본 주기 | 월 1회 재선정 (홀딩 필요 시 분기) |
| 변동 표시 | 신규 편입 / 유지 / 제외 + 변동 사유 명시 |
| 리포트 갱신 | 월 1회 기본 + 실적 발표/대형 이벤트 시 수시 갱신 |
| 긴급 변동 | 즉시 Short List 제외/교체 + **어드민(사용자 본인)에게 이메일/텔레그램 알림** |

### 10.7 비용

> **[Tier 2/3 사용자 노출 제거에 따라 재계산 필요]** 기준선 단순화 후 Tier 2/3는 내부 파이프라인 산출물로 격하됨. Tier 1(30개) 기준 추정치는 별도 산정 예정. 아래 숫자는 기존 추정치(참고용).
